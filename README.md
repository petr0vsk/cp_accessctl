# Check Point Access Control (cp_accessctl)

Утилита на Python для автоматизированного аудита, извлечения и экспорта параметров удаленного доступа (VPN / Remote Access) пользователей из конфигурации Check Point Management API.

## 📌 Проблема и контекст

В корпоративных правилах межсетевого экрана Check Point для предоставления доступа обычно используются не конкретные пользователи, а высокоуровневые объекты — **Access Roles (Роли доступа)**. 

Сложность заключается в том, что Роль доступа является лишь "контейнером" и не хранит в себе персональные ограничения по времени подключения. Кроме того, администраторы часто отключают доступ не путем удаления пользователя, а отключая само правило (переводя его в статус `Disabled`).

Этот скрипт решает обе проблемы: он проходит по всей иерархии объектов ("сверху вниз") и сверяется с реальным состоянием боевых правил фаервола, чтобы выдать точную матрицу фактических доступов.

## ⚙️ Логика работы (Гибридный подход)

Программа выполняет аудит в несколько этапов:
1. **Поиск объектов:** Скрипт ищет все роли доступа по заданному корпоративному паттерну (например, `user_ar_ra-*`) с помощью `show-objects`.
2. **Проверка активности (Аудит правил):** Через `where-used` и `show-access-rule` проверяется, привязана ли роль к какому-либо правилу и **включено ли (Enabled)** это правило в данный момент.
3. **Парсинг расписаний:** Извлекаются объекты Time, повешенные на правило, с расшифровкой дат их действия (`show-time`).
4. **Выгрузка профилей:** Скрипт раскрывает группы пользователей внутри роли и вытаскивает индивидуальные часы работы каждого сотрудника (`from-hour`, `to-hour`, `connect-daily`).
5. **Экспорт (ETL):** Данные денормализуются (сводятся в плоскую структуру) и сохраняются в формате CSV.

## 📊 Формат выгружаемых данных (CSV)

Результатом работы скрипта является файл `vpn_users_export.csv` (кодировка `UTF-8-SIG`, разделитель `;`), готовый для загрузки в базы данных или открытия в Excel. 

Каждая строка файла соответствует **одному конечному пользователю**.

| Столбец (Поле) | Описание | Пример значения |
| :--- | :--- | :--- |
| **`access_role`** | Имя родительской роли доступа (контейнера). | `user_ar_ra-filkina` |
| **`user_name`** | Имя (логин) конечного пользователя. Является первичным ключом. | `VPN_FilkinaUE` |
| **`user_group`** | Группа пользователей (если пользователь состоит в группе). | `user_gr_ra-filkina` |
| **`access_status`** | Фактический статус доступа (зависит от включенного правила). | `Active` или `Disabled` |
| **`profile_days`** | Разрешенные дни недели из свойств профиля пользователя. | `Daily`, `Mon,Tue`, `Any` |
| **`profile_start_time`** | Время начала доступа (из профиля). | `08:00` |
| **`profile_end_time`** | Время окончания доступа (из профиля). | `18:00` |
| **`rule_time_objects`** | Имена объектов Time, прикрепленных к правилу (через запятую). | `Filkina_schedule` |
| **`rule_valid_from`** | Дата начала действия правила (по объекту Time). | `2024-11-01` или `Always` |
| **`rule_valid_to`** | Дата окончания действия правила. | `2024-11-30` или `Never` |

## 📈 Использование с BI-системами и Базами данных

Плоская структура выходного файла оптимизирована для подхода **Star Schema (Звездная схема)**. Для построения полноценной витрины данных для отдела Информационной Безопасности (ИБ) рекомендуется:

1. Использовать `vpn_users_export.csv` как **таблицу фактов**.
2. Получить из Active Directory / 1С **таблицу измерений** (справочник `hr_users.csv` с полями: `user_name`, `ФИО`, `Отдел`, `Руководитель`).
3. Соединить их `JOIN` запросом по ключу `user_name` в любой аналитической СУБД (например, легковесной **DuckDB**) для вывода на дашборды (Apache Superset, Metabase, PowerBI).

**Пример SQL-запроса для витрины DuckDB:**
```sql
SELECT 
    hr.ФИО,
    hr.Отдел,
    vpn.access_status AS Статус,
    vpn.profile_start_time AS Начало_работы,
    vpn.profile_end_time AS Конец_работы
FROM read_csv_auto('vpn_users_export.csv') AS vpn
LEFT JOIN read_csv_auto('hr_users.csv') AS hr 
    ON vpn.user_name = hr.user_name
WHERE vpn.access_status = 'Active';
