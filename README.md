# Утилита для извлечения параметров доступа пользователей Check Point

Данный проект представляет собой Python-скрипт, использующий Check Point Management API для автоматизированной выгрузки настроек времени удаленного подключения (VPN/Remote Access) для конкретных пользователей.

## 📌 Проблема и контекст

В корпоративных правилах межсетевого экрана Check Point (Access Control Rules) для предоставления доступа обычно используются не конкретные пользователи, а высокоуровневые объекты — **Access Roles (Роли доступа)**[cite: 98]. 

Сложность заключается в том, что Роль доступа является лишь "контейнером"[cite: 99]. [cite_start]Она связывает воедино сети, устройства и группы пользователей, но не хранит в себе персональные ограничения по времени подключения[cite: 100]. [cite_start]Если вы попытаетесь запросить Роль доступа через команду для пользователей (`show-user`), API вернет ошибку `generic_err_object_not_found`, так как это абсолютно разные классы объектов в базе данных[cite: 101].

Настройки разрешенного времени (такие как `connect-daily`, `from-hour`, `to-hour`, `connect-on-days`) являются встроенными атрибутами исключительно стандартного объекта **User (Пользователь)**[cite: 102]. 

Чтобы узнать, в какое время разрешено подключаться сотруднику, привязанному к определенной роли, необходимо пройти по всей иерархии объектов "сверху вниз". Эту задачу и автоматизирует данный скрипт.

## ⚙️ Логика извлечения информации (Как это работает)

Программа выполняет цепочку последовательных API-запросов, спускаясь по дереву объектов Check Point:

### Шаг 1: Запрос Роли доступа (`show-access-role`)
Скрипт отправляет запрос к Management API, передавая имя искомой роли (например, `user_ar_ra-filkina`)[cite: 105]. 
В полученном JSON-ответе скрипт ищет блок `users`. [cite_start]На этом этапе мы обычно получаем не конечного пользователя, а привязанную **Группу пользователей (User Group)** (например, `user_gr_ra-filkina`)[cite: 106].

### Шаг 2: Раскрытие Группы пользователей (`show-user-group`)
Так как ограничения времени не хранятся и на уровне группы, скрипт берет UID найденной группы и выполняет команду `show-user-group`[cite: 107].
Из ответа извлекается массив `members` — список всех участников данной группы.

### Шаг 3: Запрос данных конечного пользователя (`show-user`)
Скрипт перебирает участников группы. [cite_start]Найдя объект с типом `user`, программа берет его UID и выполняет финальный запрос `show-user`[cite: 110]. Именно этот запрос обращается к персональной карточке сотрудника.

### Шаг 4: Извлечение и форматирование параметров времени
Из итогового JSON-объекта пользователя скрипт извлекает нужные ключи[cite: 111]:
* `from-hour` — время начала доступа (например, "00:00")[cite: 111].
* `to-hour` — время окончания доступа (например, "23:59")[cite: 112].
* `connect-daily` — флаг (true/false), указывающий на ежедневный доступ[cite: 113].
* `connect-on-days` — массив конкретных дней недели (если доступ не ежедневный)[cite: 114].

Полученные данные форматируются и выводятся в консоль в удобном, человекочитаемом виде.

## 🚀 Особенности реализации
**Надежность сессий:** Реализован механизм `try...finally`, который гарантирует, что даже в случае ошибки или отсутствия объекта, скрипт вызовет команду `logout()` и не оставит "висящих" сессий на Management сервере[cite: 116].
**ООП подход:** Взаимодействие с API инкапсулировано в классе `CheckPointClient`, что облегчает поддержку и масштабирование кода[cite: 117].
**Соответствие PEP 8:** Код отформатирован по стандартам Python (включая использование `@staticmethod` для вспомогательных функций логики вывода)[cite: 118].
